# Add --version subcommand to agent_pool and gsd

## Motivation

When using package manager auto-detection (pnpm dlx, npx, etc.), we need to ensure gsd uses the same version of agent_pool that it was built with. Version mismatches could cause subtle bugs.

Currently there's no way to:
1. Query the version of a binary
2. Ensure gsd and agent_pool versions match when using dlx commands

## Proposed Solution

### 1. Generate version.txt during CI

During CI build, before publishing npm packages, generate a `version.txt` file in the artifacts folder containing the version being published.

**In `.github/workflows/ci.yml`, add to the publish job:**

```yaml
- name: Write version files
  run: |
    VERSION="${GITHUB_REF#refs/tags/v}"
    if [[ ! "$GITHUB_REF" =~ ^refs/tags/v ]]; then
      VERSION="0.0.0-main-${GITHUB_SHA::8}"
    fi
    echo "$VERSION" > libs/agent_pool/artifacts/version.txt
    echo "$VERSION" > libs/gsd/artifacts/version.txt
```

### 2. Embed version in binary at compile time

Use a build script to read version.txt (if present) and embed it via environment variable.

**Create `crates/agent_pool_cli/build.rs`:**

```rust
fn main() {
    // Check for version.txt in the expected location relative to this crate
    // When built during npm publish, version.txt will be at ../../libs/agent_pool/artifacts/version.txt
    let version_paths = [
        "../../libs/agent_pool/artifacts/version.txt",
        "../version.txt",
        "version.txt",
    ];

    for path in version_paths {
        if let Ok(version) = std::fs::read_to_string(path) {
            let version = version.trim();
            println!("cargo:rustc-env=AGENT_POOL_VERSION={}", version);
            return;
        }
    }

    // Fallback: use Cargo.toml version
    println!("cargo:rustc-env=AGENT_POOL_VERSION={}", env!("CARGO_PKG_VERSION"));
}
```

### 3. Add --version flag to CLI

**In `crates/agent_pool_cli/src/main.rs`:**

```rust
const VERSION: &str = env!("AGENT_POOL_VERSION");

#[derive(Parser)]
#[command(name = "agent_pool")]
#[command(version = VERSION)]
#[command(about = "Agent pool for managing workers with file-based task dispatch")]
struct Cli {
    // ...
}
```

Clap automatically handles `--version` and `-V` flags when you add `#[command(version = ...)]`.

### 4. Add Version subcommand (optional)

For programmatic access, add a subcommand that outputs just the version:

```rust
#[derive(Debug, Subcommand)]
enum Command {
    /// Print version information
    Version {
        /// Output as JSON
        #[arg(long)]
        json: bool,
    },
    // ... other commands
}
```

Output:
```bash
$ agent_pool version
0.1.0

$ agent_pool version --json
{"version": "0.1.0"}
```

### 5. GSD uses matching version with dlx

When gsd needs to invoke agent_pool via dlx, it should request the same version:

```rust
// In resolve_agent_pool_invocation()
let gsd_version = env!("GSD_VERSION");

AgentPoolInvocation::PackageManager {
    program: "pnpm".to_string(),
    prefix_args: vec![
        "dlx".to_string(),
        format!("@gsd-now/agent-pool@{}", gsd_version),
    ],
}
```

This ensures that `gsd@0.1.0` always uses `agent_pool@0.1.0`, not whatever `latest` happens to be.

## Version Format

- Tagged releases: `0.1.0`, `1.2.3`
- Main branch builds: `0.0.0-main-abc12345`

Both gsd and agent_pool will use the same format.

## Open Questions

1. **Build script location** - Should the build script look for version.txt relative to the crate, or use an environment variable set by CI?

2. **Fallback behavior** - If version.txt isn't found, should we:
   - Use Cargo.toml version (current proposal)
   - Use "dev" or "unknown"
   - Fail the build

3. **Version mismatch handling** - When gsd detects agent_pool version mismatch, should it:
   - Warn and continue
   - Error and exit
   - Silently continue

## Testing

1. `agent_pool --version` prints version
2. `agent_pool version` prints version
3. `agent_pool version --json` prints JSON
4. Version matches between gsd and agent_pool when using dlx
5. Local development (no version.txt) falls back to Cargo.toml version
